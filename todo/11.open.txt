# Description

Implement four additional F-BASIC functions that are useful in a web IDE context:

## Functions to Implement

### 1. CSRLIN
- **Description**: Returns the current cursor line (Y position)
- **Syntax**: `CSRLIN`
- **Returns**: Integer 0-23 (current cursor line)
- **Reference**: Manual page 87

### 2. POS
- **Description**: Returns the current cursor column (X position)
- **Syntax**: `POS(n)` - n is a dummy argument (always 0 in most BASICs)
- **Returns**: Integer 0-27 (current cursor column)
- **Reference**: Manual page 85

### 3. SCR$
- **Description**: Reads a character from the screen at specified position
- **Syntax**: `SCR$(X, Y)`
- **Returns**: Single character string at position (X, Y)
- **Reference**: Manual page 87

### 4. BEEP
- **Description**: Produces a beep sound
- **Syntax**: `BEEP`
- **Returns**: None (produces sound)
- **Reference**: Manual page 80
- **Implementation**: Use Web Audio API oscillator

## Implementation Steps

For each function:

1. **Parser** (`src/core/parser/parser-tokens.ts`):
   - Add token (e.g., `Csrlin`, `Pos`, `Scr`, `Beep`)

2. **Parser** (`src/core/parser/FBasicChevrotainParser.ts`):
   - Add parsing rule for function/statement

3. **Function Evaluator** (`src/core/evaluation/FunctionEvaluator.ts`):
   - Add evaluation methods for CSRLIN, POS, SCR$
   - Track cursor position in screen state

4. **Sound Device** (for BEEP):
   - Add beep method to sound device
   - Use Web Audio API

5. **Tests**:
   - Create tests in `test/evaluation/` or `test/executors/`

## Acceptance Criteria

- [ ] CSRLIN returns current cursor line
- [ ] POS returns current cursor column
- [ ] SCR$ reads character from screen buffer
- [ ] BEEP produces audible beep sound
- [ ] All functions have tests
- [ ] Documentation updated
